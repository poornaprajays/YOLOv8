What Every Engineering Student Should Know

A structured deep-dive into Computer Vision, CNNs, and YOLOv8 — written for people who want to understand the why, not just run the code.

Computers don't "see" like humans. Every image is just a 3D array of numbers — shape (height, width, 3) where each pixel is an RGB triplet from 0–255.

How a CNN extracts meaning from pixels

A Convolutional Neural Network learns to detect patterns by sliding small matrices (kernels) across the image. Each kernel fires when it sees its pattern:

Layer 1–3
Edges, gradients, corners
Layer 5–10
Textures, patterns (fur, scales, fabric)
Layer 15–25
Parts (wheels, eyes, doors, legs)
Layer 30+
Whole objects (car, dog, person)
# A 3×3 edge-detecting Sobel kernel (CNNs LEARN values like these)
kernel = [[-1,  0,  1],
          [-2,  0,  2],   # Fires strongly on vertical edges
          [-1,  0,  1]]

# In PyTorch, the convolution operation:
output = F.conv2d(image, weight=kernel, padding=1)
💡 Key insight: You don't hand-craft these kernels. Backpropagation + gradient descent finds the optimal values automatically from millions of training examples.
Classification
Q: What is in this image?
A: "dog"

One label per whole image. Solved well since AlexNet (2012).

→
Object Detection
Q: What is here, and where?
A: [("dog", 0.94, [120,80,380,420]), ...]

Multiple boxes + labels + scores. YOLO solves both simultaneously.

Detection is harder because the model must solve localization (where?) and classification (what?) together. Before YOLO, pipelines ran in two stages — slow. YOLO's breakthrough: one forward pass solves both.

Backbone (C2f)
Extracts feature maps at 3 scales: 80×80, 40×40, 20×20. Large grids detect small objects; small grids detect large objects.
↓ feature maps
Neck (FPN + PAN)
Feature Pyramid Network merges features top-down (coarse → fine). Path Aggregation Network adds bottom-up path. Rich multi-scale representation.
↓ fused features
Head (Decoupled · Anchor-Free)
Two separate branches: regression (predicts bounding boxes) + classification (predicts class probabilities). No anchor templates needed.
↓ raw predictions
NMS (Non-Max Suppression)
If two boxes overlap by more than the IoU threshold (~50%), keep only the highest-confidence box. Removes duplicate detections of the same object.
💡 Anchor-free explained: YOLOv1–5 needed pre-defined "anchor boxes" — fixed aspect-ratio templates. If your objects didn't match these templates, accuracy suffered. YOLOv8 predicts distances from grid-cell centers directly. Simpler, faster to train, more accurate.
IoU
Overlap ÷ Union
How much does the predicted box cover the true box? 0 = no overlap, 1.0 = perfect. A detection is "correct" if IoU ≥ 0.5.
Precision
TP ÷ (TP + FP)
Of all our predictions, how many were actually correct? High precision → few false alarms.
Recall
TP ÷ (TP + FN)
Of all real objects, how many did we find? High recall → few missed objects.
mAP50-95
Mean AP @ [0.5:0.95]
Industry-standard. Averaged over all 80 COCO classes AND IoU thresholds 0.50 to 0.95 in steps of 0.05. YOLOv8n = 37.3.
⚖️ The Precision–Recall Trade-off

Lowering confidence threshold → higher recall (find more objects) but lower precision (more false positives). This is controlled by the confidence slider in this app. Try it!

Training YOLOv8 from scratch on COCO required 8× NVIDIA A100 GPUs for days. You don't need to do that ever.

How transfer learning works in practice
  1. Take yolov8n.pt — weights from training on 118,000 COCO images
  2. The model already knows edges, textures, object parts, shapes
  3. Add your own dataset (even 100–500 labeled images)
  4. Fine-tune: only the last layers re-learn (takes hours, not days)
# Fine-tune YOLOv8 on your own custom dataset
from ultralytics import YOLO

model = YOLO("yolov8n.pt")  # Start from pre-trained weights

results = model.train(
    data="my_dataset.yaml",  # Your classes + image paths
    epochs=50,
    imgsz=640,
    batch=16,
)
# Done — now model detects YOUR custom objects
💡 Why it works: Low-level features (edges, textures) are universal — a wheel in a car photo looks the same as in a bicycle photo. High-level, class-specific knowledge is what you're fine-tuning. You're literally "transferring" general visual knowledge to your specific task.
